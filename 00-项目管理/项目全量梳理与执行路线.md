项目的创作核心已经从旧版 V1 升级到 V2；但生产链仍停在 V1。下一步不是继续堆素材，而是先把“剧本 → 提示词 → 素材 → 配音 → 剪辑 → 字幕 → 输出”的链路统一为 V2，再开始正式制作。

## 4. 关键风险与处理优先级

| 优先级 | 风险 | 后果 | 处理 |
|---|---|---|---|
| P0 | V2 剧本与字幕、合成程序不一致 | 剪辑会做出错误版本 | 09—12 已同步；重建字幕和 V2 合成流程后再出片。 |
| P0 | “6/10”文件由 AI 生成文字 | 数字、文字或签名错乱，证据链失真 | 用统一后期道具替换文字，所有镜头重复使用。 |
| P0 | 直接复用 V1 成片或一键拼接脚本 | 叙事、时长、结尾均与 V2 冲突 | V1 移入历史参照；新建 V2 剪辑工程。 |
| P1 | 01—05、07 为旧版创作过程资料 | 被误用于制作会导致剧情回退 | 已在文件头标记为历史档案；参赛材料 13、14 已同步 V2。 |
| P1 | 无真实配音和音效资产 | 15 分钟成片无法完成节奏控制 | 粗剪锁定后录制/生成配音，再配环境声。 |
| P2 | 合成脚本路径写死旧设备 | 换电脑无法复现输出 | V2 输出阶段改成项目相对路径。 |
| P2 | 未从 PDF 逐条核验征集规则 | 格式、时长、版权或 AI 说明可能踩线 | 参赛前人工核对通知原文并形成清单。 |

## 5. 唯一正确的执行顺序

### 阶段 1：统一制作依据（先做，零生成额度）

1. 09、10 已改为 V2 的关键帧与短视频提示词。
2. 11、12 已改为 V2 的声音/剪辑脚本。
3. 01—05、07 已标记为历史档案；13、14 已同步 V2。
4. 已建立 V2 字幕台词底稿；粗剪锁时长后再生成 SRT。

**阶段完成标准：** 06、08、09、10、11、12 已可共同用于 V2 制作；接下来需要同步对外材料并重建字幕。

### 阶段 2：生成与验收（按免费额度）

按 `15.低额度素材生成计划.md` 生成并登记：

1. A1：6/10 桌面三人关键帧。
2. A2：第一次犹豫后签字。
3. A3：第二次快速签字。
4. A4：小周写“6”。

**阶段完成标准：** 4 项均有原片、选中版、台账状态；文件文字以统一后期道具替换。

### 阶段 3：后期制作

1. 在 `剪辑工程/01-项目文件/` 新建 `就这一次-V2.0-粗剪`。
2. 先用静帧、A1—A4 和现有可复用视频搭建 12—15 分钟无配音粗剪。
3. 锁画面顺序后，在 `音频素材/01-配音/` 制作配音；环境音优先于配乐。
4. 依据锁定时长制作全新 SRT，存入 `剪辑工程/02-字幕文件/`。
5. 精剪、调色、审看后输出到 `剪辑工程/03-输出暂存/`；送审文件才进入 `成品输出/`。

### 阶段 4：参赛交付

1. 同步更新作品说明、答辩说明、剧照和 AI 创作说明。
2. 核对征集通知中的格式、时长、分辨率、署名、版权和 AI 作品要求。
3. 对照本文件第 1 节的“不可改动事实”审看成片与全部材料。
